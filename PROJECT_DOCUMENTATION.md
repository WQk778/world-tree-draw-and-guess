# AI 你画我猜智能互动游戏网站 - 项目文档

## 1. 项目概述
本项目是一个基于 Web 的多人在线“你画我猜”互动游戏平台，创新性地引入了 **AI（OpenAI Vision）** 作为裁判和提示生成者。用户可以在房间内进行绘画创作，AI 负责识别画作内容并生成生活化的提示词，其他用户根据提示进行竞猜。系统还包含完整的积分排行、房间管理及后台管理功能。

## 2. 技术架构

### 2.1 技术栈
- **前端**: Vue.js (Vue 3) + Vite
- **后端**: Express.js (Node.js)
- **数据库 & 基础设施**: Supabase
  - **Auth**: 用户认证
  - **Database**: PostgreSQL (存储业务数据)
  - **Storage**: 存储用户画作图片
  - **Realtime**: 实现房间状态和游戏进程的实时同步
- **AI 服务**: OpenAI Vision API (GPT-4-Vision / GPT-4o)

### 2.2 架构图示
- **客户端 (Vue)**: 处理 Canvas 绘图、游戏交互界面、实时订阅。
- **服务端 (Express)**: 处理 AI 接口转发、复杂业务逻辑（如积分结算、定时任务）、管理员 API。
- **数据层 (Supabase)**: 提供基础 CRUD API、文件存储和 WebSocket 实时能力。

## 3. 功能需求详情

### 3.1 普通用户核心功能（5 大模块）

#### 1. 管理房间互动
- **创建房间**: 生成唯一的 6 位房间码，设定人数上限（2-10人）。
- **加入房间**: 通过输入房间码进入房间。
- **查看成员**: 实时显示房间内成员列表、昵称、角色（房主/玩家）及在线状态。

#### 2. 管理绘画创作
- **Canvas 绘画**: 提供 8 种基础颜色、3 档画笔粗细、撤销与清空功能。
- **提交画作**: 将 Canvas 内容转为图片上传至 Supabase Storage，并记录数据。

#### 3. 管理 AI 提示查看
- **AI 识别**: 画作提交后自动触发 AI 识别。
- **提示生成**: 将 AI 识别结果转化为生活化的短语提示（非直接答案）。
- **同步显示**: 所有猜词玩家界面同步显示 AI 生成的提示。

#### 4. 管理猜词提交
- **输入答案**: 玩家输入猜测的词汇。
- **验证与结算**: 系统对比答案，若正确则记录猜对状态并实时更新积分。

#### 5. 管理积分排行
- **个人积分**: 查看当前用户的总积分及历史明细。
- **全站排行**: 查看全站用户积分排行榜，显示前 100 名及个人排名。

### 3.2 管理员核心功能（4 大模块）

#### 1. 管理 AI 参数配置
- **模型配置**: 切换 OpenAI 模型版本（如 gpt-4-vision-preview）。
- **阈值调整**: 设置 AI 识别的置信度阈值。
- **提示模板**: 编辑用于生成提示词的 Prompt 模板。

#### 2. 管理积分规则
- **规则配置**: 新增或修改积分获取规则（如：猜对+10分，连胜+5分）。
- **规则开关**: 启用或禁用特定积分规则。

#### 3. 管理超时房间
- **超时设定**: 配置房间无操作的自动销毁时间（如 10 分钟）。
- **自动清理**: 系统自动扫描并清理超时房间，释放资源。

#### 4. 管理数据报表
- **核心统计**: 统计用户活跃度（DAU/MAU）、AI 识别成功率等。
- **报表导出**: 生成并导出 Excel 格式的运营报表。

## 4. 数据库设计 (Schema: public)

系统共包含 11 张核心表，满足 3NF 设计规范，支持多对多复杂业务关系。

### 4.1 用户与房间
- **profiles**: 用户档案（昵称、头像、总分），扩展自 `auth.users`。
- **rooms**: 房间信息（房间码、房主、状态）。
- **room_members**: [多对多] 房间与用户的关联，记录角色和在线状态。

### 4.2 游戏核心
- **game_rounds**: 游戏回合（题目、当前画手、状态）。
- **drawings**: 画作数据（图片 URL、AI 原始响应、转换后的提示）。
- **guess_records**: [多对多] 用户对画作的猜词记录（是否猜对、得分）。

### 4.3 积分与配置
- **point_rules**: 积分规则定义（规则名、分值）。
- **point_records**: [多对多] 用户积分流水，关联触发规则。
- **ai_configs**: AI 模型与参数配置表。

### 4.4 数据分析
- **rankings**: 排行榜快照表（定期计算写入）。
- **login_records**: 用户登录日志（用于活跃度统计）。

## 5. 接口与集成

### 5.1 Supabase 集成
- 使用 `supabase-js` 客户端进行前端数据读写和实时订阅。
- 开启 Row Level Security (RLS) 保护用户隐私数据。

### 5.2 AI 接口 (Express 中转)
- **Endpoint**: `POST /api/ai/recognize`
- **Input**: Image URL, Room Context
- **Process**: 调用 OpenAI API -> 解析 JSON -> 存入数据库 -> 推送客户端。

### 5.3 管理员接口
- 需要特定的 Admin Role 权限验证。
- 提供对 `point_rules` 和 `ai_configs` 的写权限。
